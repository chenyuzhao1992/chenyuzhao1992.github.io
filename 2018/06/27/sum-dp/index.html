Definition

? what’s the key

key is get relationship between subproblems

? how find the key

just guessing all possible, then pick the best
helps by drawing call trees and DAG

? DP is

careful Brute Force, optimizing problems into polynominal time O(n), by reusing subproblems f(i)

? how find subproblems

Subproblems for String

  • suffixes A[i:] - O(n)
  • prefixes A[:i] - O(n)
  • substring A[i,j] - O(n^2)

5 Steps

  1. Define Subproblems
  2. Guess part of solution
  3. Relate subproblems solution
  4. recursive & memoization OR build DP table bottom-up
  5. Solve original problem

Base Case / Termination

this is the key, otherwise stackoverflow

Memo

  • int[] or boolean[]
  • better with HashMap?
    • when 0 also is valid as the default value for int[] is 0
    • when not continuous but sparse mapping, so saving memory
  • reduce to constant?
    • only use related state/subproblems
    • use existing variables

Helper function

  • parameters with the state of the subproblems/functions
    • such as, helper(int[][] map, int x, int y) for f(x, y) in the map

Variations

  • Dimensions: 1D -> 2D
    • depend on memo, like int[] or int[][]
  • State Transferring Formula: adjusted from naive adding
    • complex or derivation required, such as f(i) = (k - 1) * (f(i - 1) + f(i - 2))
    • more terms, such as f(i) = f(i-1) + f(i-5) + f(i-10) + f(i-25)
  • Options: parallel
    • several “return” cases when “top->bottom”
    • or several paralleling update when “bottom->top”
