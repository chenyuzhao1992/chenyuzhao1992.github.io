How to Prepare for Interviews

Austin Zhao - Meaningful & Neat Coding
2018-05-30

Process

0. Exploring & Locating & Practicing & Writing

Exploring

  • searching via Linkedin, glassdoor

Locating

  • target company & positions
  • required skill stack
  • building connections

Practicing

  • Coding
    • paper and high-level first, structuring via pseudocode, presenting, testing cases
    • computer just for entering
  • Learning / Courses
  • Presenting & Communicating
  • Projects

Writing

  • Resume
  • Linkedin
  • Personal Blog & Github & Twitter
  • Cover Letter

1. Applying

Submitting

  • referal first and best, via friends, Linkedin
  • on-personal, like career fair
  • online

Type

  • Campus CLN
  • Career Fair
  • Official Website, for target and big company
  • Linkedin, for others

2. Interviewing

Type of Interviewer

  • Recruiter / Eternal
  • HR / Internal

xx-Oriented

  • 60s Self-Introduction
    • [TBD]
  • Behavioral-Oriented
    • [TBD]
  • Resume-Oriented
    • basic information, like graduation time, targeted positions
    • course, what done and how related
    • projects, how related
  • Courses-Oriented
    • computer networking
    • operating systems
    • database
  • Coding-Oriented
    • type
      • audio/video, via phone/app, using online docu
      • on-site, with team leader, using paper/white board
    • cover
      • tree: dfs, bfs
      • algorithms: dp, backtracking, greedy, divide and conquer
      • data structure: hash, stack, queue
      • searching: binary, sorting: all kinds of
      • logic: iterative, recursive
      • topics: bit, math, string and list manipulating
      • more? : graph(union-find)

Communicating

  • clear information, such as content, interviewers, tools
  • follow up
    • thanks
    • track results, after 1~2 weeks
  • build connections
    • adding Linkedin

3. Handling Offers

  • accept, negotiation
  • reject

How to do Interview

How Algorithm Problems are Organized

  • Classical: core, the feature
  • Follow up / Variation:
    • yes or not: boolean, first reach then first back
    • accumulating / extending : from 1 to n
    • optimal: min, max
    • printing / collecting: steps, results
